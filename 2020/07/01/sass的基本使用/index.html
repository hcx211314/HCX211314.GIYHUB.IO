  1. 什么时候声明变量
    • 该值至少出现了2次。
    • 该值至少可能会被更新一次。
    • 该值所有的表现都与变量有关（非巧合）

    1. 避免选择器嵌套

      • 选择器最大的问题是将最终的代码难以阅读。开发者需要巨大精力计算不同缩进级别下的选择器具体的表现效果。
      • 选择器越具体则声明语句越冗长，而且对最近选择器的引用（&）也越频繁。在某些时候，出现混淆选择器路径和探索下一级选择器的错误率很高，这非常不值得。

    • 无默认值

      @mixin border-radius($radius:5px){
      -webkit-border-radius:$radius;
      border-radius:$radius;
      }

      @mixin box-shadow($shadow…){
      @if length($shadow)>=1{

      @inclue prefixer(box-shadow, $shadow);

      }@else{

      $shadow:0 0 4px ragb(0, 0, 0, 0.3);
@inclue prefixer(box-shadow, $shadow);

      }
      }

      @mixin border-radius($radius:5px){
      -webkit-border-radius:$radius;
      border-radius:$radius;
      }

      button{
      @include border-radius;
      }

      div{
      @inclue border-radius(10px);
      }

      div{
      -webkit-border-radius:10px;
      border-radius:10px;
      }

      @mixin center($width, $height){
      width: $width;
      height: $height;
      position: absolute;
      top: 50%;
      left: 50%;
      margin-top: -($height) / 2;
      margin-left: - ($width) / 2;

      }

      .box-shadow{
      @include center(500px, 300px)
      }

      .box-shadow{
      width: 500px;
      height: 100px;
      position: absolut;
      top: 50%;
      left: 50%;
      margin-left: -150px;
      margin-top: -250px;
      }

      .btn{
      border:1px solid #ccc;

      font-size:14px;

      .btn-primary{
      background-color:#666;

      @extend: .btn;

      通过@extend调用的占位符，编译出来的代码会将相同的代码合并到一块，会让css文件更加干净。

  • 混合宏VS继承VS占位符

    1. Sass中混合宏的使用
      编译出来的CSS清晰告诉了大家，他不会合并相同的样式代码，如果样式文件中调用同一个混合宏，会产生对各对应的样式代码，造成代码冗余。但是其可以传参数。
      如果代码块中涉及到变量，建议使用混合宏来创建。
    2. Sass中继承
      在使用继承后，编译出来的css会将使用继承的代码块合并到一起，通过组合
